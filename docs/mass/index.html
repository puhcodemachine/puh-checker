> МАССОВАЯ ЗАГРУЗКА И ПРОВЕРКА · Режим А

Загрузи .txt (каждая сид-фраза — отдельной строкой). Каждая сразу станет отдельной задачей с числовым именем (1, 2, 3…) и проверится по всем путям. Авто-перепроверка раз в 24ч полным кругом. Это отдельная среда — её задачи не пересекаются с главной панелью (хранилище IndexedDB).

Проверяется как в Режиме А: BTC (Legacy/P2SH/SegWit), LTC ×3, DOGE, DASH, ETH (Standard+Ledger на ETH/BSC/Polygon), ETC — ~45 адресов на сид. Очередь рандомизируется; число потоков подстраивается само (1→5): при большой очереди и здоровых API растёт, при лимитах/ошибках снижается и откатывается — без сбоев. Ретраи с задержкой, ротация нод/ключей (свои ключи эксплореров — в KEYS в mass.js). Среда изолирована (IndexedDB «puh_mass»), главная панель не затрагивается.
